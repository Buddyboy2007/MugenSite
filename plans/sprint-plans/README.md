# Sprint Planning

This directory contains sprint planning documents, backlogs, and retrospectives.

## Sprint Structure

Each sprint follows this format:

```
sprint-plans/
├── sprint-01/          # Current sprint
│   ├── backlog.md     # Sprint backlog and tasks
│   ├── goals.md       # Sprint goals and objectives
│   └── retrospective.md # Sprint retrospective
├── sprint-02/          # Next sprint
└── templates/          # Template files for planning
```

## Sprint Planning Process

1. **Backlog Grooming** - Review and prioritize user stories
2. **Capacity Planning** - Assess team availability and capacity
3. **Task Breakdown** - Break down stories into specific tasks
4. **Estimation** - Estimate effort for each task
5. **Sprint Goal** - Define clear, achievable sprint goals
